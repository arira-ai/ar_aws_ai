# 1️⃣ **Amazon Bedrock** (Most Important)

Think of **Bedrock** as:
✨ *“The place where you use ready-made AI models like ChatGPT.”*

### What Bedrock provides:

* **Foundation Models (FMs)**:
  Claude, Llama, Amazon Titan → you can use them directly.
* **Text**: Chatbots, content writing, summarization.
* **Images**: Generate or understand images (via FM partners).
* **Embeddings**: Convert text→numbers for search/recommendations.
* **Agents**: AI that can perform tasks by calling APIs.
* **Prompt Management**: Store, version, test prompts.
* **Model Evaluation**: Check accuracy, quality, bias.
* **Guardrails**:
  Content filtering → prevent harmful/unsafe outputs.

### Simple Example:

“Create a customer support chatbot” → Use **Bedrock + Claude**.

# 2️⃣ **Amazon SageMaker** (High Level Only)

Think of SageMaker as:
🛠️ *“A workshop to build, train, tune, and deploy ML models.”*

You don’t need deep details — only basic concepts.

### What SageMaker includes:

* **Studio** → Main ML interface (IDE for ML).
* **Notebooks** → Jupyter notebooks for coding ML.
* **Training** → Train your own ML model.
* **Inference** → Deploy model endpoints for predictions.
* **Autopilot** → Automated ML: you give data; it trains best model.
* **JumpStart** → Use prebuilt ML models (NLP, vision, etc).
* **Feature Store** → Store reusable ML features (age, salary, etc).
* **Pipelines** → Automate ML workflows.

### Simple Example:

You want to create your **own custom fraud detection model** → Use SageMaker.

# 1️⃣ **AMAZON BEDROCK (VERY IMPORTANT)**

Bedrock is AWS’s **generative AI platform**.
You do **NOT train models here** — you only **use / customize** foundation models.

---

## 🔹 **Foundation Models (FMs)**

These are ready-made LLMs hosted by AWS.

### Main models:

* **Anthropic Claude** → Best for reasoning, summarizing, coding.
* **Meta Llama** → Efficient, open model, cheaper.
* **Amazon Titan** → Amazon’s own models (text, embedding).

**Exam Tip:**
*If the question says “use GenAI without managing infrastructure,” answer = Bedrock.*

---

## 🔹 **What you can do in Bedrock**

### **1. Text Models**

For:

* Chatbots
* Summaries
* Content generation
* Q&A
* Writing assistance

**Example exam question:**
“Business wants multilingual summarization with no model training.”
→ *Use Bedrock Text Models (Claude/Llama/Titan).*

---

### **2. Image Models**

For:

* Image generation
* Editing
* Style transformation

**Exam Tip:**
If user wants an image from text → *Bedrock Image Models* (Stable Diffusion).

---

### **3. Embeddings**

Convert text → numeric vector for:

* Search
* Recommendations
* Similarity matching

**Exam Tip:**
*Embedding = When question mentions “semantic search”, “similar text,” or “content matching.”*

---

### **4. Bedrock Agents**

Agents connect LLMs with **real actions** like calling APIs.

**Example:**
“LLM automatically checks order status in ERP.”

**Exam Tip:**
*Use Agents when the model must perform tasks, not just answer.*

---

### **5. Prompt Management**

Features:

* Save prompts
* Version prompts
* Compare outputs
* Trace history

**Exam Tip:**
Prompt management = *Prompt orchestration* → part of Bedrock.

---

### **6. Model Evaluation**

Evaluate model quality based on:

* Accuracy
* Hallucinations
* Bias
* Response quality

---

### **7. Guardrails**

Ensure safe, filtered output.

* Block profanity
* Block hate, self-harm
* Block PII
* Restrict topics
* Customize allowed content

**Exam Tip:**
If the question asks for *content control*, answer: **Bedrock Guardrails**.
