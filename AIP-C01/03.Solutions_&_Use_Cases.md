# ✅ **Domain 3 — Solutions & Use Cases (25%)**

This domain tests whether you know **which AWS AI/ML service solves which business problem**.
You’re not expected to design complex systems — just know **“which service to pick for what use case.”**

Below is a clean breakdown, in **simple terms**, with **exam-style clarity**.

---

# 🔶 **1. Matching AI Services to Business Problems (MOST important)**

### 🟦 **1. Call Center Transcription → Amazon Transcribe**

* Converts **speech → text**
* Used for: call-center recordings, meeting notes, voice logs.

**Exam Clue:** “audio files,” “transcribe calls,” “convert customer calls to text.”

---

### 🟩 **2. Multi-Language Website → Amazon Translate**

* Translate text between languages automatically.

**Exam Clue:** “localize website,” “convert English to Spanish,” “multi-language app.”

---

### 🟧 **3. Product Recommendations → Amazon Personalize**

* Recommends **products, videos, music** to users.
* Similar to Netflix/Amazon shopping recommendations.

**Exam Clue:** “recommendation engine,” “next best product,” “personalized suggestions.”

---

### 🟨 **4. Image Classification for Defects → Rekognition / Lookout Vision**

* **Rekognition** → general computer vision (faces, objects, labels).
* **Lookout for Vision** → industrial defect detection (scratches, dents).

**Exam Clue:**

* Manufacturing defects → **Lookout for Vision**
* Detect objects/labels → **Rekognition**

---

### 🟪 **5. Build Custom LLM App → Amazon Bedrock**

Use to:

* Generate text
* Build chatbots
* Create agents that automate workflows
* Use foundation models (Claude, Llama, Titan)

**Exam Clue:** “LLM,” “gen AI,” “agents,” “RAG,” “embeddings,” → always **Bedrock**.

---

### 🟫 **6. Forecasting Sales → Amazon Forecast**

* Predicts **future values** like sales, inventory, demand.

**Exam Clue:**
“time-series,” “predict next week,” “inventory planning.”

---

### 🟪 **7. Chatbot for Customer Support → Lex + (optional) Bedrock**

* **Lex** handles the conversation (voice & text bot).
* **Bedrock** helps with advanced understanding.

**Exam Clue:**

* IVR system? → **Lex**
* Natural conversation? → **Lex + Bedrock**

---

# 🔶 **3. Scaling ML Workloads (Very short exam-style explanation)**

### 🟦 **Use SageMaker for scalable training/inference**

* Distributed training
* Several instance types
* Endpoints automatically scale

### 🟩 **Use Batch Transform for large offline predictions**

* When you must run inference on huge datasets.

### 🟧 **Use async & multi-model endpoints to save cost**

* Many models deployed on one endpoint = cheaper.

# 🔶 **4. Cost-Effective Selection of Services**

### ✔ Fully-managed AI services = cheapest and easiest

(example: Rekognition, Comprehend, Textract).

### ✔ Choose Bedrock over training custom models

* Saves GPU cost
* No infrastructure needed

### ✔ Use Spot instances for training

* Up to 90% cheaper.

### ✔ Serverless analytics (Athena) to save storage + compute costs.

# 🔷 **5. Basic Architecture Patterns**

Only high level needed for the exam:

### 🟦 **Data → S3**

Primary storage for ML.

### 🟨 **Prepare → Glue DataBrew** or Glue ETL

Simple data cleaning.

### 🟩 **Train → SageMaker**

For custom ML.

### 🟧 **Use AI → Bedrock or AI Services**

Text, image, speech, chatbots.

### 🟪 **Monitor → CloudWatch**

Track metrics & logs.
