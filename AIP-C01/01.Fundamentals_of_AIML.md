# 🧠 **Domain 1 – Deeper Explanation (Exam-oriented)**

---

## **1️⃣ Artificial Intelligence (AI)**

AI is a broad field where machines are designed to **perform tasks that normally need human intelligence**.

### What AI covers:

* Decision-making
* Understanding language
* Recognizing objects
* Solving problems
* Learning from experience

**Example:**
Alexa understanding your voice and giving the right response = AI.

---

## **2️⃣ Machine Learning (ML)**

ML is a **subset of AI** where systems learn from **data**, not hard-coded rules.

### Key idea:

* You give *examples* → ML picks patterns → applies them to new situations.

**Example:**
Train a spam filter using thousands of “spam” and “not spam” emails.

---

## **3️⃣ Deep Learning (DL)**

Deep Learning is a **subset of ML** that uses **artificial neural networks with many layers**.

### Why deep?

Each layer extracts more complex features:

* Layer 1 → edges
* Layer 2 → shapes
* Layer 3 → objects

**Example:**
Face recognition, self-driving cars, image understanding → mostly DL.

---

# 🔄 **Training vs Inference**

---

## **4️⃣ Training**

Training is the process where the model **learns patterns** from data.

### What happens during training:

* Model sees examples
* Adjusts weights
* Minimizes error

**Example:**
Giving 100,000 images to teach a model how to recognize cats.

---

## **5️⃣ Inference**

Inference is when the trained model is **used to make predictions**.

### What happens in inference:

* You pass *new data*
* Model gives an answer instantly
* No learning happens here

**Example:**
You upload a new image → model predicts “Cat”.

AWS focuses on both concepts because training often happens in SageMaker, while inference happens in Bedrock, SageMaker Endpoints, etc.

---

# 🎓 **Supervised vs Unsupervised vs Reinforcement Learning**

---

## **6️⃣ Supervised Learning**

Uses **labeled data** → every input has the correct output.

### Used for:

* Classification (spam/not spam)
* Regression (house price prediction)

**Example:**
Image labeled “Dog”, “Cat”, “Car”.

---

## **7️⃣ Unsupervised Learning**

Uses **unlabeled data** → model finds structure on its own.

### Used for:

* Clustering customers
* Finding patterns
* Dimensionality reduction

**Example:**
Grouping customers into “high spenders”, “casual buyers”.

---

## **8️⃣ Reinforcement Learning (RL)**

A model learns by **trial and error** with reward/penalty.

### Used for:

* Robotics
* Game AI
* Industrial optimization

**Example:**
A robot tries different ways to walk → rewarded when it performs correctly.

---

# 🔢 **Common Algorithms (High-Level)**

---

## **9️⃣ Linear Models**

The simplest ML models — use a **line or plane** to make predictions.

### Used for:

* Predicting continuous values (price)
* Simple classification problems

**Example:**
House price = (size × weight) + bias

Why AWS wants this:
These models are cheap, fast, and used inside Forecast, SageMaker, etc.

---

## **🔟 Decision Trees**

A model that makes decisions by asking a series of **yes/no questions**.

### Features:

* Easy to interpret
* Works with numbers and text
* Forms the basis for Random Forest, XGBoost

**Example:**
Is age > 30?
→ Yes → Has high income?
→ No → Reject loan

---

## **1️⃣1️⃣ Clustering Algorithms**

Group similar items without labels.

### Common types:

* K-Means
* DBSCAN

**Used for:**

* Customer segmentation
* Anomaly detection
* Market analysis

**Example:**
Grouping customers based on shopping behavior.

---

## **1️⃣2️⃣ Convolutional Neural Networks (CNNs)**

Special deep learning networks used for **images and video**.

### Why CNNs?

* They detect patterns like edges, textures, shapes
* Extremely accurate for vision tasks

**Used in:**

* Rekognition
* Lookout for Vision
* Self-driving systems

---

## **1️⃣3️⃣ Recurrent Neural Networks (RNNs)**

Used for **sequential data** — text, audio, time series.

### Why RNNs?

They remember **previous inputs**.

**Used in:**

* Chatbots
* Speech recognition
* Language models (older generation)

Today RNNs are replaced by **Transformers** (used in LLMs like Claude & GPT).

---

# 🎯 **Quick Summary for Exam**

* AI → Broad intelligence
* ML → Learn from data
* DL → Neural networks
* Training → Learn
* Inference → Predict
* Supervised → Labeled
* Unsupervised → Unlabeled
* RL → Reward-based learning
* Linear models → Simple predictions
* Trees → Rule-based decisions
* Clustering → Grouping
* CNN → Images
* RNN → Text sequences

# 🧠 **Domain 1 Concepts + AWS Services**

---

# 1️⃣ **AI, ML, Deep Learning → Which AWS Service?**

## **Artificial Intelligence (AI)**

AWS gives ready-made AI through **AI Services** (no ML expertise needed).

### AWS services representing AI:

* **Amazon Rekognition** → image & video intelligence
* **Amazon Comprehend** → text intelligence (sentiment, entities)
* **Amazon Textract** → document intelligence
* **Amazon Translate** → language translation
* **Amazon Lex** → conversational AI
* **Amazon Transcribe** → speech-to-text

➡️ All these are “AI” because they perform intelligent tasks like humans.

---

## **Machine Learning (ML)**

ML on AWS = **Amazon SageMaker**.

### SageMaker allows you to:

* Train ML models
* Tune ML models
* Deploy ML models
* Use built-in algorithms
* Use pre-trained models (JumpStart)

➡️ SageMaker = AWS’s primary ML platform.

---

## **Deep Learning (DL)**

DL = neural networks, GPU training, LLMs.

### AWS DL services:

* **Amazon SageMaker** (GPU training, Deep Learning Containers)
* **AWS Inferentia / Trainium** chips
* **Amazon Bedrock** (hosting Deep Learning foundation models like Claude, Llama)

➡️ Bedrock = the easiest way to use Deep Learning/LLMs without coding.

---

# 2️⃣ **Training vs Inference → AWS Services**

## **Training**

Training = teaching the model using data.

### AWS services for training:

* **SageMaker Training Jobs**
* **SageMaker Studio**
* **SageMaker Autopilot** (automatically trains ML models)

**Example:**
Training a fraud detection model → use SageMaker.

---

## **Inference**

Inference = model predictions.

### AWS services for inference:

* **SageMaker Endpoints** (real-time prediction)
* **SageMaker Batch Transform** (batch prediction)
* **Amazon Bedrock** (LLM inference only — no training)

**Example:**
Sending a new image to Rekognition → inference (prediction).

---

# 3️⃣ **Supervised, Unsupervised, Reinforcement → AWS mapping**

## **Supervised Learning → AWS**

Used when data has labels.

### AWS tools/services:

* **SageMaker built-in algorithms** (Linear Learner, XGBoost)
* **Ground Truth** for data labeling
* **SageMaker Autopilot** (auto supervised learning)

**Example:**
Predict house prices → SageMaker Linear Learner.

---

## **Unsupervised Learning → AWS**

Used for finding structure in unlabeled data.

### AWS tools:

* **SageMaker K-Means**
* **SageMaker PCA**
* **Amazon OpenSearch Anomaly Detection**

**Example:**
Cluster customers → SageMaker K-Means.

---

## **Reinforcement Learning → AWS**

AWS supports RL mainly through SageMaker.

### AWS tools:

* **SageMaker RL** (prebuilt RL toolkits: Ray RLlib, Coach)

**Example:**
Train a delivery robot → use SageMaker RL.

CNNs are mostly used for image tasks.

AWS implementation:

* **Lookout for Vision** for industrial defect detection

Used for:

* Image classification

* **SageMaker Deep Learning Containers** (TensorFlow/PyTorch RNNs)
* **Amazon Transcribe** (speech sequence modeling)
* **Amazon Comprehend** (text sequence modeling)

But modern AWS services use **Transformers, not RNNs**.
