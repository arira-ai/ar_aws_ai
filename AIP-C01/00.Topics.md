# ✅ **AWS Certified AI Practitioner – Short Notes**

### **What this certification proves**

* You understand **AI/ML fundamentals**.
* You know how to use **Amazon AI/ML services**.
* You can identify the **right AWS AI tool** for a given business use case.
* You can evaluate **responsible AI, governance, and security** basics.
* No coding or deep ML knowledge required — entry level.

---

# 📘 **Syllabus Synopsis (as per AWS official blueprint)**

AWS splits the exam into **4 major domains**:

---

## **1️⃣ Domain 1 — Fundamentals of AI/ML (20%)**

**You must know:**

* What is **Artificial Intelligence**, **Machine Learning**, **Deep Learning**.
* Training vs inference.
* Supervised vs unsupervised vs reinforcement learning.
* Common algorithms (high level):

  * Linear models, trees, clustering, CNNs, RNNs.
* Data basics:

  * Data collection, labeling, cleaning, bias sources.
* Foundational GenAI concepts:

  * LLMs, embeddings, tokenization.

**Focus is only conceptual, no maths or coding.**

---

## **2️⃣ Domain 2 — AWS AI/ML Services (40%)**

**Know which service is used for what. This is the largest domain.**

### **Amazon Bedrock (very important)**

* Foundation models (Claude, Llama, Titan).
* Text, images, embeddings, agents.
* Prompt management, model evaluation.
* Guardrails (content filtering & safety).

### **Amazon SageMaker (high-level only)**

* Studio, Notebooks, Training, Inference, Autopilot.
* JumpStart for prebuilt models.
* Feature Store, Pipelines (conceptual).

### **AI Services (Fully managed — must memorize!)**

* **Vision:** Rekognition, Textract, Panorama
* **Language:** Comprehend, Translate, Transcribe, Polly
* **Chatbot:** Lex
* **Search:** Kendra, OpenSearch ML features
* **Personalization:** Personalize, Forecast
* **Industrial AI:** Lookout for Metrics/Equipment/Vision

### **Analytics + Data**

* Glue DataBrew, Glue ETL basics
* Athena, Redshift ML (concept only)
* Data lakes (S3 + Lake Formation)

---

## **3️⃣ Domain 3 — Solutions & Use Cases (25%)**

**You must know which AWS AI service solves which business problem.**

Examples:

* Call center transcription → **Transcribe**
* Multi-language website → **Translate**
* Product recommendations → **Personalize**
* Image classification for defects → **Rekognition / Lookout Vision**
* Build custom LLM app → **Bedrock + Agents**
* Forecasting sales → **Forecast**
* Chatbot for customer support → **Lex + Bedrock**

Also:

* Scaling ML workloads.
* Cost-effective selection of services.
* Basic architecture pattern (no deep design knowledge needed).

---

## **4️⃣ Domain 4 — AI Governance, Security, and Responsible AI (15%)**

**Key things to know:**

* Bedrock **Guardrails** (very important)
* Data privacy and encryption
* IAM basics for AI workloads
* Avoiding model bias
* Explainability (SHAP, feature importance — conceptual)
* Model monitoring (drift, bias, data quality)
* Shared responsibility model for AI
